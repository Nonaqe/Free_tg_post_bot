# 📰 Telegram-бот: Планировщик новостей

🌐 [English](README.en.md) · **Русский**

Бот мониторит приватные Telegram-каналы-**источники**, копит новые посты в очередь и
публикует их в каналы-**получатели** — по расписанию, по слотам или вручную с модерацией.
Всё управление — через inline-кнопки в личной переписке с ботом.

> Стек: **Python 3.11+**, [aiogram 3.x](https://docs.aiogram.dev), SQLite (aiosqlite),
> APScheduler, loguru.

---

## ✨ Возможности

- **Источники и получатели** — несколько каналов, у каждого получателя свои настройки
  (задержка, теги, префикс/суффикс, медиа-фильтр).
- **Очередь-листалка** — посты пролистываются по одному, с реальным превью медиа
  (фото/видео/документ/альбом), редактированием текста и времени.
- **Гибкая публикация**:
  - по расписанию (приход + задержка),
  - по слотам времени (09:00 / 15:00 / 21:00 — редактируются),
  - вручную «сейчас» во все каналы или в выбранные чекбоксами,
  - **ручная модерация** — посты ждут, пока человек не решит, когда публиковать.
- **Тихие часы**, активные дни недели, минимальный интервал, лимит постов в сутки.
- **Часовой пояс** — всё расписание считается в выбранном TZ.
- **Роли**: `superadmin` / `editor` / `viewer`.
- **Статистика** с полосками и экспортом в CSV, лог действий админов.
- **Уведомления**: ошибки отправки, «очередь пуста N часов», ежедневный отчёт, рестарт.
- **Надёжность**: защита от дублей (атомарный захват поста), авто-восстановление после
  сбоя, ежедневный бэкап БД, повтор при ошибках, обработка FloodWait.
- **Автоочистка** служебных сообщений бота в личке (кроме последнего).

## 🚀 Установка

```bash
git clone <url-репозитория>
cd TGNEWS

python -m venv .venv
.venv\Scripts\activate          # Windows
# source .venv/bin/activate     # Linux/macOS

pip install -r requirements.txt
cp .env.example .env            # затем заполнить (см. ниже)
```

### Настройка `.env`

Обязательные поля:

| Переменная           | Описание                                                        |
|----------------------|-----------------------------------------------------------------|
| `BOT_TOKEN`          | Токен бота от [@BotFather](https://t.me/BotFather)              |
| `SOURCE_CHANNEL_IDS` | ID каналов-источников через запятую (бот должен быть админом)   |
| `ADMIN_IDS`          | ID администраторов через запятую. **Первый** = `superadmin`     |

Остальные параметры (задержка, интервал, webhook и т.д.) — опциональны, см. комментарии
в `.env.example`.

### Запуск

```bash
python -m bot.main
```

В Telegram открой личку с ботом → `/start` → появится меню.

## 🛠 Как пользоваться

1. Добавь бота **администратором** в канал-источник (иначе он не видит посты).
2. Добавь бота админом с правом **Post Messages** в каналы-получатели.
3. В боте: 📡 **Каналы** → ➕ Добавить → вставь `@username` или `chat_id` получателя.
4. Публикуй пост в источнике — он попадёт в очередь.
5. Дальше по вкусу:
   - оставить как есть → уйдёт по расписанию;
   - 📋 **Очередь** → ✏️ Время / 🚀 Сейчас → выбрать момент или отправить сразу;
   - включить ⚙️ **Настройки → Ручная модерация** → посты будут ждать твоего решения.

### Команды

- `/start`, `/menu` — главное меню (дашборд)
- `/queue` — очередь
- `/pause`, `/resume` — глобальная пауза отправок

## 📦 Деплой

### Docker (рекомендуется)
```bash
cp .env.example .env   # заполнить
docker compose up -d --build
```
БД и бэкапы — в `./data` (переживают пересборку), авто-рестарт включён.

### systemd (VPS без Docker)
```bash
sudo cp deploy/tgnews-bot.service /etc/systemd/system/
sudo systemctl daemon-reload && sudo systemctl enable --now tgnews-bot
journalctl -u tgnews-bot -f
```

### Webhook
Задай в `.env` `WEBHOOK_BASE_URL=https://домен` (нужен публичный HTTPS).
Пусто → работает polling.

```
bot/
├── main.py              точка входа: бот + БД + планировщик
├── config.py            чтение и валидация .env

├── services/            queue / sender / filter / notify
├── db/                  модели и репозиторий (aiosqlite)
└── utils/               media, logger, timeutil, autodelete
```
